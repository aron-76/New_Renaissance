# AI Native Quant 思路 — 市场模拟

## 一、核心思路

用多个 Agent 模拟真实市场，每个 Agent 代表一类人群（而非一个个体），通过所有 Agent 的订单流叠加，涌现出完整的市场行为。

## 二、目的：决策

市场模拟的目的是为了辅助交易决策。基本思路：

- 同样的当前市场状态 + 同样的 Agent 配方，跑多次模拟
- 每次因为微观层面的随机性（注意力采样、订单到达顺序、成交时序），走出不同的价格路径
- 统计这些路径的分布

决策参考（简化描述，实际需要更精细的框架）：
- 概率分布整体偏正 + 期望收益为正 → 考虑做多
- 概率分布分散 → 降低仓位或不做
- 还需要考虑赔率（上涨幅度 vs 下跌幅度），不能仅看涨跌概率

这不是预测某一条具体路径，而是生成未来价格的概率分布，在分布上寻找正期望的位置。

### 配方的价值

Agent 配方好不好，看的是概率校准（calibration）：
- "我说 60% 概率涨的股票，实际上大约 60% 涨了吗？"
- "我说 5% 分位是 1680，实际跌破 1680 的大约是 5% 的情况吗？"

一个好的配方 = 一个校准良好的概率分布生成器。这一点需要在大量历史数据上验证。

## 三、Agent 设计（设计提案）

- Agent 不是一个人，而是一类市场参与者（如散户追涨型、机构价值型、量化高频型、游资短线型、做市商型等，具体分类待研究）
- Agent 的输出可以理解为一个**订单矩阵**（品种轴 × 时间轴），代表该群体在一段时间内的整体行为
- 所有 Agent 的订单矩阵叠加后，形成完整的订单流，喂入撮合引擎

注："订单矩阵"是本文提出的概念，实际上是稀疏的订单列表，概念上覆盖全品种 × 全时间。这一设计是否合理需要实验验证。

## 四、分层架构（设计提案）

Agent 和撮合引擎之间存在巨大的频率差距，可能需要多层中间架构逐级放大：

| 层级 | 名称 | 频率（估算） | 职责 |
|------|------|-------------|------|
| L4 | Agent (LLM) | 分钟级（具体待定） | 输出行为参数（方向、强度、参与率） |
| L3 | 订单生成器 | 秒级 | 根据行为参数 + 盘口数据，统计采样生成具体订单 |
| L2 | 订单路由/风控 | 逐笔 | 合规检查，按品种分发 |
| L1 | 撮合引擎 | 逐笔（事件驱动） | 订单簿维护，价格优先+时间优先撮合 |

注：各层的频率是初步估算，实际参数需要通过实验确定。Agent 的调用频率涉及成本和模拟真实性的权衡。

### 关键认知

- 撮合引擎是事件驱动的，有订单来才撮合，不是按固定时间间隔运转
- 订单簿不是撮合引擎的输入，而是它内部维护的状态
- 撮合引擎的核心输入是订单指令（新订单、撤单、改单等）
- 不同股票的成交频率差异很大：活跃股可能每秒多笔，冷门股可能数分钟一笔（具体数据需要从实际行情中统计）

## 五、信息层（设计提案）

Agent 的决策依赖实时信息，信息获取本身也需要模拟真实性。

### 三层信息源

1. **公共信息源**：所有 Agent 都能看到（宏观数据、公告、突发事件）
2. **个性化信息源**：推荐算法 + 社交圈子 + 持仓相关推送
3. **主动搜索**：Agent 根据自身兴趣定时/随机搜索信息

### 注意力调度器（待验证的设计）

核心假设：不同类型的市场参与者获取信息的节奏不同，这种差异会影响订单流的时间分布。

可能的建模方式（参数均为初步设想，需要从实际数据中校准）：
- 散户型：信息获取碎片化，可用泊松过程建模
- 机构型：开盘和尾盘关注度高，盘中可能有空白期
- 量化型：近实时系统化获取
- 重大事件可以"打断"正常节奏，提前触发决策

不同群体的信息获取时间差 → 反应速度差 → 价格分阶段演变。这一假设是否成立以及如何量化，需要与真实市场的微观数据对比验证。

## 六、涌现性

模拟系统如果有效，其有效性可能建立在 Agent 之间不可解耦的基础上。它们呈现的是一个共同的涌现结果，不能简单地通过观察单个 Agent 的行为来套利。类似于天气系统中各物理场耦合在一起的情况。

同样的配方 + 同样的信息，每次模拟走出不同的路径，是因为微观层面的随机性逐步放大。这种随机性在真实市场中也存在——同样的基本面条件下，每天的走势也不完全相同。

注：涌现性是这套方法成立的关键假设之一，需要在实际模拟中验证是否能观察到合理的涌现行为。

## 八、待验证的核心假设

1. Agent 分类（散户型、机构型等）是否能有效代表真实市场的参与者结构？
2. 订单矩阵 + 统计采样能否生成与真实订单流统计性质相似的数据？
3. 注意力调度器的参数能否从真实数据中校准？
4. 多次模拟产生的概率分布是否具有良好的校准性？
5. 整个系统能否产生合理的涌现行为？
