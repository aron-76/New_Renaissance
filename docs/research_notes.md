# AI Native 量化框架研究笔记

## 一、项目调研

### 1.1 TwinMarket —— 多智能体金融市场仿真

**论文**：TwinMarket: A Scalable Behavioral and Social Simulation for Financial Markets（NeurIPS 2025，ICLR 2025 最佳论文）

**核心思路**：用 LLM Agent 模拟投资者行为，研究微观个体行为如何汇聚成宏观集体动力学（如泡沫、衰退）。

**架构**：
- 微观层：BDI 认知框架（Belief 信念 → Desire 欲望 → Intention 意图）
- 市场环境：订单驱动交易系统 + 撮合引擎（价格优先→时间优先→最大成交量）
- 社交环境：动态社交网络 + 信息推荐系统

**每日模拟流程**：
1. 加载数据（新闻、交易日历、用户策略、初始信念）
2. 构建社交图（基于交易相似度）
3. 并发处理每个 Agent（刷论坛 → 阅读新闻 → 推荐股票 → 检索信息 → 更新信念 → 选股 → 交易决策 → 发帖）
4. 撮合引擎执行交易 → 更新帖子评分
5. 日期+1，循环

**人设体系**：
- 1000 个 Agent，998 个独立 sys_prompt
- 用户类型：大V (11)、小博主 (77)、普通股民 (912)
- 投资策略：技术面 (600) vs 基本面 (400)
- 6 个行为维度（处置效应、彩票偏好、分散投资度、总收益、换手率、交易频率）× 2-3 档 = 332 种行为组合
- 信念初始化态度：乐观 40%、悲观 50%、中性 10%

**对人设体系的评价**：
- 维度选择合理（行为金融核心变量）
- 取值分布有问题（过多均匀分布，真实散户的处置效应和换手率应严重右偏）
- 缺少关键角色：机构投资者、做市商、游资、量化基金
- 缺少维度：年龄、资金量、教育/职业
- 行为偏差是静态标签，真实世界中会随市场环境动态变化

**对建模有效性的评价**：
- 论文目标不是预测市场，而是社会科学仿真工具
- 涌现现象是"半自然、半预设"的——随机交易者的动量策略是硬编码的，行为偏差是通过 prompt 注入的
- 价格发现机制被简化（目标价格在昨收附近随机生成，不是 LLM 决定的）
- 所有 Agent 用同一个 LLM，思维模式本质相似
- 存在循环论证风险：注入偏差 → 观察到偏差 → 声称验证了理论
- 缺乏反事实验证：不注入偏差时现象是否还会出现？

### 1.2 FinSight —— 多智能体金融研报生成

**论文**：FinSight: Towards Real-World Financial Deep Research

**核心思路**：多智能体系统自动生成接近人类分析师水平的多模态金融研报（文字 + 图表 + 引用）。

**三阶段流水线**：
1. 数据收集：Deep Search Agent（多轮搜索+网页浏览） + Multi-Source Data Collection Agent（金融API+宏观API）
2. 数据分析：多轮代码交互式分析 + 迭代视觉增强（LLM生成图表→VLM评审→反馈修改→循环）
3. 报告生成：两阶段写作（先生成 Chain-of-Analysis 精简段落 → 再扩展为完整报告）

**核心技术 CAVM（Code Agent with Variable Memory）**：
- 将数据、工具、Agent 统一到可编程变量空间 V = V_data ∪ V_tool ∪ V_agent
- Agent 通过生成并执行代码来操作变量空间，比 function calling 更灵活

**实验结果**：总分 8.09，显著超越 Gemini Deep Research (6.82) 和 OpenAI Deep Research (6.11)，可视化得分 9.00 远超所有竞品

**生成统计**：每份报告平均 62,586 tokens、51.2 张图表、983 次搜索、470 个页面浏览

**评价**：
- 报告粒度是券商深度研报级别（20页+），不适合日频信息汇总
- 生成耗时长，不适合日报
- 但 CAVM 架构可以拆开复用

### 1.3 MiroFish —— 群体智能预测引擎

**仓库**：https://github.com/666ghj/MiroFish （AGPL-3.0）

**核心思路**：输入真实数据（新闻、报告），生成数千个有独立人格和记忆的 AI Agent，让它们在模拟世界中交互，根据涌现行为产出预测报告。

**关键依赖 - Zep**：
- Zep 是 AI Agent 的记忆服务（知识图谱 + 长期记忆）
- 在 MiroFish 中的作用：
  1. 构建实体知识图谱（人物、公司、事件→节点；关系→边）
  2. 从图谱生成 Agent profile（给每个 Agent 注入不同的"认知"）
  3. 动态更新图谱（Agent 行为回写到图谱，形成闭环）
- Zep 不是图形化工具，是 Agent 的大脑/记忆层
- 替代方案：Neo4j + embedding 可以自建

## 二、关键技术概念

### 2.1 Embedding（向量嵌入）

把文字变成一串数字（向量），让计算机能算"两段话有多像"。

用途：语义搜索。搜"半导体"能匹配到"芯片"和"集成电路"，而不只是关键词匹配。

TwinMarket 的做法：用 Qwen3-Embedding-0.6B 模型（通过硅基流动 API）将新闻转为向量，存入 FAISS 向量库，Agent 查询时做相似性检索。

两种方式：调 API（简单，要钱）vs 本地跑模型（免费，需要算力）。

### 2.2 关键词提取（语义输入 → 关键词输出）

和 embedding 不同，这是从文本中提取关键词：

- jieba：中文分词 + TF-IDF，最经典
- KeyBERT：语义关键词提取，效果最好，本地运行不需要 API
- YAKE：纯统计方法，最快

### 2.3 数据获取

**akshare**：完全免费、开源（MIT），不需要 API key。直接从东财、新浪、同花顺公开接口抓数据。覆盖 A 股行情、财务报表、行业板块、新闻、宏观数据等。

**真正需要爬的只有新闻和研报**，其他结构化数据都有 akshare。

自己爬的最简方案：duckduckgo_search + requests + BeautifulSoup，零成本。难点在反爬对抗和持续维护。

## 三、核心思考：AI Native 量化的方向

### 3.1 核心观点

传统量化公司买卡、挖因子、招 PhD，是军备竞赛。大模型 API 背后已经包含了最顶尖的算法和海量算力，可以直接基于大模型做决策。

但 LLM Agent 做高频交易不可能（API 一次调用几百毫秒到几秒），Agent 的战场在日频/周频以上的信息密集型策略——比的不是谁快，而是谁理解得更深。

### 3.2 两条技术路线

**路线1：Agent 配方仿真**
- 用 1000 个 Agent 画像模拟 3 亿真实投资者的群体行为
- 按资金量和行为特征分层（散户/游资/公募/外资/量化等）
- 如果模拟走势匹配真实走势 → 配方有效 → 可以预测
- 问题：LLM 的训练数据包含历史市场信息，回测存在数据泄露风险
- 验证方式：只能通过虚拟盘前瞻验证，无法传统回测
- 定位：长期探索方向

**路线2：Agent 挖因子**
- Agent 从研报、论文、新闻、社交媒体中发现因子
- 因子是数学公式或打分，输出是数字
- 数字可以用传统方式回测（无数据泄露）
- MiroFish 作为因子之一：模拟预测事件走向 → 对标的打分 → 变成 mirofish_factor
- 定位：短期可用，能立刻产出价值

**两条路线的关系**：路线2 包含路线1。MiroFish 仿真不再是独立的预测系统，而是因子工厂里的一台机器。准不准由 IC 值（信息系数）自动判断。

### 3.3 LLM 回测的数据泄露问题

传统策略是纯函数（同样的输入一定得到同样的输出），LLM 是黑盒（脑子里有整个互联网的知识）。让 LLM 分析 2023 年的茅台，它可能"记得"后来跌了，而不是真的在推理。

**你没法区分 LLM 的输出是"真推理"还是"回忆"。**

应对方案：
- 匿名化回测：隐去股票名、日期、公司名，只给结构化特征
- 训练截止日之后的数据做验证
- 真实前瞻预测：每天跑预测，次日对比，积累真实记录
- 最终解法：Agent 输出因子公式（纯数学），因子回测不依赖 LLM，没有泄露问题

### 3.4 Agent 挖因子的两层价值

**第1层：更快** —— 传统因子也能挖，但速度快 100 倍。读研报→提公式→写代码→回测，全自动。

**第2层：更宽** —— 能挖出传统方法无法构造的因子：
- 文本情绪因子（财报电话会管理层语气评分）
- 政策敏感因子（某行业被点名次数）
- 舆情因子（社交媒体情绪极性）
- 跨文档关联因子（供应商出负面→本公司受影响）
- 逻辑推理因子（原材料涨价+不能提价=毛利率下降）

第2层是真正的护城河，结构化因子已被挖尽，非结构化信息因子是蓝海。

### 3.5 非结构化信息如何量化为因子

所有因子最终都是一个数字。非结构化信息通过 LLM 打分量化。

事件因子的完整链条：
1. 采集：每天自动获取新闻/公告/MiroFish预测
2. 筛选：Agent 过滤出重要事件
3. 匹配：每个事件 → 识别相关标的（同一事件对不同标的影响不同）
4. 打分：事件×标的打分矩阵
5. 合成：每个标的的综合事件因子 = 所有相关事件分数的加权和
6. 输出：一个数字，和 PE、动量因子一样使用

单次打分可能不准，但统计上有区分度就够了。IC 哪怕只有 0.03，也能赚钱。

### 3.6 框架定位

一个 AI 驱动的因子发现引擎：

```
输入源：                    因子生产线：              输出：
研报/论文          →   Agent 提炼公式因子    →
新闻/公告          →   Agent 打分因子        →    因子库
社交媒体           →   Agent 舆情因子        →   （每个标的每天
MiroFish 模拟      →   MiroFish 因子         →    一组数字）
用户的灵感         →   Agent 翻译成公式      →
                                                  ↓
                                              传统回测（无数据泄露）
                                              筛选有效因子（IC/IR）
                                                  ↓
                                              因子组合
                                                  ↓
                                              交易信号
```

## 四、量化交易框架调研

### 4.1 现有框架对比

| 框架 | 协议 | 定位 | 状态 |
|------|------|------|------|
| vnpy | MIT | 量化交易平台（回测+实盘） | 活跃 |
| backtrader | GPL-3.0 | 纯回测库 | 已停更 |
| zipline | Apache-2.0 | 事件驱动回测 | 原版停更 |
| freqtrade | GPL-3.0 | 加密货币交易机器人 | 活跃 |
| MiroFish | AGPL-3.0 | 群体智能预测 | 活跃 |

### 4.2 vnpy 回测原理

本质是事件回放：逐根 K 线重放历史数据 → 策略看到 K 线后决定买卖 → 下一根 K 线撮合 → 计算收益。

传统策略能回测是因为策略是纯函数（只依赖你喂给它的数据）。LLM Agent 不行（它脑子里有整个互联网的知识）。

### 4.3 是否基于 vnpy 开发

不建议。vnpy 架构老（2015年设计），和 AI/LLM 完全无关，强行嫁接会很别扭。

建议：独立开发 AI 层，需要实盘时通过 API 对接 vnpy 或券商接口。网关不是技术难点（几天就能对接），是资质和合规门槛。

## 五、开源协议

项目选择 AGPL-3.0 协议：
- 别人用了代码必须开源，防止白嫖
- 连 SaaS 部署也要开源
- 可以做双授权商业化（开源免费/闭源付费）
- 和 MiroFish（AGPL-3.0）完全兼容
- TwinMarket（MIT）可以被 AGPL 项目使用
- 代价：大公司通常回避 AGPL，社区采用率低于 MIT/Apache
